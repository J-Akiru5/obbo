However, it is missing critical quality infrastructure: no CI/CD pipeline, no pre-commit hooks, no formatter, minimal tests, and several architectural inconsistencies. The remediation plan spans 5 phases with 22 actionable items.

Critical Architecture Issues

  • critical
    Dual Database Approach — Prisma Client Dead Code
    Prisma is installed, configured, and generates a full client (14 models, ~2,200 lines) at src/generated/prisma/. However, zero source files import from it. All runtime DB access uses Supabase. This creates schema drift risk, unnecessary build complexity, and confusion.
    prisma/schema.prisma + src/generated/prisma/ (unused)
  • critical
    No CI/CD Pipeline
    Zero CI/CD workflows. No build validation, lint checks, test runs, or deployment automation. All quality gates are manual. Risk of broken code reaching production.
    .github/workflows/ (missing)

✅ Note: The proxy.ts file is actually correct for Next.js 16 — the middleware.ts convention was deprecated and renamed to proxy.ts with a named proxy export. No rename needed.

🟡 High (4)

  • H-1
    30+ Dark Mode Bugs
    Hardcoded light-mode colors (bg-white, text-gray-*, bg-amber-*, etc.) render invisible or broken in dark mode across admin and client portals.
    30+ files — see last audit report
  • H-2
    Accessibility: Viewport Blocks Zoom (WCAG 1.4.4)
    maximum-scale:1 + userScalable:false disables pinch-to-zoom on all pages. Violates WCAG Success Criterion 1.4.4.
    src/app/layout.tsx:21-22
  • H-3
    Touch Targets Below 44px
    Buttons (32px default), inputs (32px), select triggers (32px), quantity selectors (32px) all fail the 44px minimum touch target guideline.
    src/components/ui/button.tsx, input.tsx, select.tsx, etc.
  • H-4
    Critical Test Coverage Gap
    Only 2 test files with 14 assertions. No tests for auth, API routes, server actions, or any page components. No E2E tests.
    Entire project

🔵 Medium (8)

  • M-1
    No Prettier / Formatter
    No code formatting standard. Every developer's editor settings produce different output.
  • M-2
    No Husky / Pre-commit Hooks
    No quality gates before commits. Broken/lint-error code can be committed freely.
  • M-3
    No lint-staged Configuration
    Even with Husky, there's no staged-file linting — entire project would be re-linted on every commit.
  • M-4
    ESLint Config Too Minimal

    eslint.config.mjs
  • M-5
    Prisma Migrations Directory Missing
    prisma/migrations/ does not exist. Cannot use prisma migrate for schema versioning.
    prisma/
  • M-6
    32 SQL Migration Files in 3 Locations
    Migrations spread across supabase/migrations/, .agents/migrations/, scripts/, and supabase/ root — no unified tracking.
  • M-7
    Schema Drift: ShipmentLedger Profit Fields

  • M-8
    Missing EditorConfig & .nvmrc
    No cross-editor formatting consistency. Node version not pinned for CI or other developers.
