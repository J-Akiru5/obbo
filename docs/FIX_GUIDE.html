Environment Setup Guide

If you're setting up the project for the first time, follow these steps.

⚙

First-Time Project Setup

Get the project running on your machine from scratch

Manual
1

Clone the repo and install dependencies

Terminal
git clone <repo-url> cd obbo npm install
2

Create your .env.local file

Copy the example and fill in your values:

Terminal
cp .env.example .env.local
3

Get your Supabase credentials

Go to supabase.com → your project → Settings → API:

  • Copy Project URL → paste as NEXT_PUBLIC_SUPABASE_URL
  • Copy anon public key → paste as NEXT_PUBLIC_SUPABASE_ANON_KEY
  • Copy service_role key → paste as SUPABASE_SERVICE_ROLE_KEY
4

Get your Database URL

Go to Supabase → Settings → Database → Connection string → URI tab:

  • Copy the connection string → paste as DATABASE_URL
  • Replace [YOUR-PASSWORD] with your actual database password
5

Get your Resend API key

Go to resend.com → API Keys → Create API Key:

  • Copy the key → paste as RESEND_API_KEY
6

Generate Prisma client and start the dev server

Terminal
npx prisma generate npm run dev
✅
Verify: Open http://localhost:3000 in your browser. You should see the landing page.
💡
Tip: If you don't have Supabase credentials yet, you can still run npm run dev — the app will start but pages that need the database will show errors. Get the credentials from your team lead.
